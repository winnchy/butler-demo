# 全天候私人管家 — 迭代评测报告

---

## 📊 v1.0 | 2026-06-03 | 双服务架构初版

### 1. 任务完成能力 — 6/10

| 子项 | 分 | 依据 |
|------|---|------|
| 餐厅推荐准确率 | 7 | restaurant_recommend 工具正常返回数据，LLM 能据此给出推荐。扣分：LLM 有时不用 API function-calling 而输出 XML 文本，导致工具未实际执行 |
| 端到端完成率 | 4 | 单个工具调用 OK，但"推荐→取号→路线→提醒→叫车→评价"全链路极少走完。LLM 逐次调用而非一次并联，流程断裂 |
| 规则遵循 | 6 | 商务不提家人、推荐给理由等规则已写入 prompt，但 LLM 偶有违反（如商务宴请推荐逛街、暴露系统用语"数据库没记录"） |
| 场景覆盖率 | 6 | 25 个工具覆盖 5 个 Skill，21 个场景脚本。但 21 个场景大多只有 1 步用户输入，未体现完整闭环 |

**改进建议**：强化 system prompt 中并联调用的强制性；增加 few-shot 示例展示完整链路。

---

### 2. 智能特性 — 4/10

| 子项 | 分 | 依据 |
|------|---|------|
| 意图解析 | 8 | 能正确识别"饿/吃/出行/穿搭/活动"等基础意图，正确路由到对应工具 |
| 工具调用稳定性 | 3 | DeepSeek 偶发 XML 文本输出而非 API function-calling，_parse_text_tools 兜底但效果打折 |
| 记忆与规划 | 4 | CHAT_HISTORY 实现多轮上下文，memory_save 工具就绪。但 LLM 不主动预判下一步，需用户逐条指令 |
| 跨 Skill 联动 | 1 | prompt 写了联动规则但 LLM 几乎不执行。推荐餐厅不自动查天气、叫车不自动设提醒 |
| 千人千面 | 2 | 三用户档案完备，但 LLM 回复风格无差异，小琴/小冉/小晴感受不到"不同管家" |

**改进建议**：在工具 description 中加入"调此工具时也应同时调 XX"的联动提示；在 USER.md 基础上追加用户语气模板。

---

### 3. 可靠性 — 7/10

| 子项 | 分 | 依据 |
|------|---|------|
| 异常处理 | 8 | LLM 不可用有降级提示；Backend 不可用有错误返回；工具超时有 try/except；Gateway 不可用自动降级 standalone |
| 错误恢复 | 7 | 文本工具解析兜底 XML；对话历史截断防 token 溢出；文件读取失败不回传错误 |
| 一致性 | 6 | CHAT_HISTORY 保证同会话上下文不丢。但跨会话/刷新后历史丢失，无持久化 |

**改进建议**：无需大幅改进，当前降级策略已较完善。可考虑会话持久化。

---

### 4. 效率 — 7/10

| 子项 | 分 | 依据 |
|------|---|------|
| 响应速度 | 6 | 简单问答 2-4s，多工具调用 8-15s。扣分：LLM 不做并联调工具导致不必要多轮 API 调用 |
| 资源占用 | 8 | 512MB 跑 OpenClaw+Python+DeepSeek，合理利用有限资源。双服务架构解耦良好 |
| Token 效率 | 6 | max_tokens=800 适中，但 system prompt 偏长（SKILL自动解析+工作流+示例），可进一步精简 |

**改进建议**：LLM 并联调用可减少响应时间 50%+；system prompt 可压到 3000 字符以内。

---

### 5. 用户体验 — 5/10

| 子项 | 分 | 依据 |
|------|---|------|
| 对话流畅度 | 5 | 整体自然，但偶有数据堆砌（如原始路线数据不加工直接输出）。Markdown **加粗** 和 --- 分割线偶有泄漏 |
| 上下文感知 | 5 | 多轮对话能记住之前推荐，但场景感知弱（工作日商务宴请推逛街、用户在公司提议回家） |
| 情绪适应 | 3 | 紧急情况语气不够温暖果断，抱怨场景未体现共情 |
| 个性化 | 4 | 用户档案卡展示良好，但对话中未体现千人千面 |
| 结构化输出 | 6 | emoji 分行部分实现，但推荐格式未严格遵循（缺商家服务行、推荐理由不充分） |

**改进建议**：输出模板用 few-shot 强化；场景标签（工作日/周末、商务/休闲）加入上下文判断；为三用户各准备语气示例。

### P0 优先修复（影响核心功能）
1. LLM 并联调用 — 一次用户消息 → 同时调多个相关工具
2. 工具调用稳定性 — 减少 XML 文本输出
3. 数据表达 — 工具结果理解后重述，不堆砌

### P1 重要修复（影响体验）
4. 千人千面 — 三用户回复风格差异化
5. Markdown 泄漏 — 彻底消除 ** 和 ---
6. 场景感知 — 商务不推逛街，工作日不提回家

### P2 锦上添花
7. 跨 Skill 联动 — 用餐带出行、穿搭带天气
8. 情绪适应 — 紧急/抱怨/开心不同语气
9. 会话持久化 — 刷新不丢历史
