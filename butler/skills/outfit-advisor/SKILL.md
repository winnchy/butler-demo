# 穿搭管家 (outfit-advisor)

## 1. 定位
成为用户身边最懂天气、最懂衣橱、最懂行程的私人形象顾问。不仅回答"今天穿什么"，更在换季、出行、突发天气等关键时刻主动关怀。

## 2. 触发条件

### 2.1 主动定时推送（由 HEARTBEAT 调度）
- **每晚 21:00**：推送次日天气预报 + 穿搭建议
- **每天早上 7:00**：推送实时天气确认 + 当日穿搭微调
- **周五 20:00**（仅当检测到换季信号且用户尚未完成衣橱调整时）：推送衣橱调整与采购提醒

### 2.2 用户被动查询
- 天气询问："今天/明天穿什么""冷不冷""要带伞吗"
- 场景穿搭："明天面试/约会/户外运动穿什么""去杭州出差带什么"
- 衣橱管理："我好像没有厚外套了""帮我看看衣橱还缺什么"

### 2.3 突发天气自动触发
HEARTBEAT 每 5 分钟轮询 `GET /api/weather/alerts`，命中以下条件立即推送紧急卡片：
- 暴雨/大风/暴雪/雷暴黄色及以上预警
- 短时强降温（24小时内 ≥8°C）
- 持续阴雨/回南天（湿度 >85% 持续 ≥2天）
- 高温橙色/红色预警
- 空气严重污染 / 紫外线极强

## 3. 执行流程与 API 调用

### 3.1 晚间推送（21:00）→ 次日预览

**Step 1**：获取用户画像
从 `life-organizer` 的 `GET /api/memory/search?user_id=xxx` 获取常驻城市、身份标签（`white_collar`/`parent`/`college_student`）、特殊偏好

**Step 2**：获取天气 → `GET /api/weather/forecast`
```json
// 返回
{
  "hourly": [{"hour": 6, "temp": 12, "condition": "多云", "rain_prob": 20}, ...],
  "daily": [{"date": "2026-06-01", "temp_high": 28, "temp_low": 18, "condition": "晴", "rain_prob": 10}, ...]
}
```

**Step 3**：获取穿搭建议 → `GET /api/outfit/suggest?user_id=white_collar`
```json
// 返回
{
  "current_temp": 18, "condition": "晴",
  "base_suggestion": "春秋过渡装",
  "recommended_items": ["长袖薄衫", "薄外套", "长裤"],
  "tip": "今日晴，气温18°C，祝您舒适"
}
```

**Step 4**：拼装晚间卡片
- 城市、日期、天气图标、高低温、体感温度
- 降雨概率、风力风向、紫外线、空气质量
- 明日逐小时简表（至少6个时点，来自 hourly）
- 未来三日概览（来自 daily）
- 穿搭建议（recommended_items + 理由）
- 温馨提醒（带伞、提前出门、收衣物等）

**Step 5**：亲子特别处理（`user_id=parent` 时）
卡片额外追加：
- 儿童当日穿搭建议（基于年龄段和活动类型）
- 儿童防晒/防雨/保暖提醒
- 儿童随身物品清单（遮阳帽、备用衣物、水壶等）

### 3.2 早晨推送（7:00）→ 当日微调

**Step 1**：获取实时天气 → `GET /api/weather/current`
```json
// 返回
{
  "temperature": 14, "feels_like": 13, "condition": "多云",
  "humidity": 55, "wind_direction": "南风", "wind_level": 2,
  "aqi": 80, "aqi_level": "良", "alerts": []
}
```

**Step 2**：获取穿搭建议 → `GET /api/outfit/suggest?user_id=white_collar`

**Step 3**：拼装早晨卡片
- 当前温度、体感温度
- 今日高低温与天气
- 逐小时预报（突出降雨开始时间，来自 forecast.hourly）
- 穿搭确认（recommended_items 转为可勾选清单）
- 通勤/出行提醒（联动 mobility-butler 获取路况）

### 3.3 突发天气处理

**Step 1**：HEARTBEAT 每 5 分钟调 `GET /api/weather/alerts`
```json
// 返回
{"alerts": [{"type": "暴雨黄色预警", "level": "黄色",
  "issued_at": "...", "expires_at": "..."}]}
```

**Step 2**：存在新预警 → 调 `GET /api/outfit/alert?user_id=xxx`
```json
// 返回
{"alerts": [{"type": "rain", "message": "突发暴雨！建议穿防水鞋、带伞，避免浅色衣物"}]}
```

**Step 3**：推送醒目预警卡片，包含：
- 天气预警详情（类型、级别、影响时段）
- 紧急行动清单（可勾选）：收衣物、关窗、调整出行
- 对出行/餐饮/日程的联动影响评估

### 3.4 换季管理

**检测机制**（每日定时推送时附带执行）：

**Step 1**：调 `GET /api/weather/trend?city=北京&days=14`
```json
// 返回
{
  "past_7d_avg_temp": 16.2,
  "future_14d_avg_temp": 22.8,
  "temp_trend": "warming",
  "is_season_shift": true,
  "current_season": "春季",
  "next_season": "夏季",
  "confidence": 0.85,
  "detail": "过去7天日均温16.2°C，未来14天预计升至22.8°C",
  "suggestion": "建议将厚棉服收纳，取出薄外套、防晒衣"
}
```

**Step 2**：判断逻辑
- `is_season_shift = true` → 进入 Step 3
- `is_season_shift = false` → 跳过

**Step 3**：检查记忆
从 `life-organizer` 读取 `wardrobe_season_ready` 标记：
- 已标记 `done` → 不再打扰
- 标记 `pending` 或不存在 → 进入提醒流程

**提醒流程**（首次检测到换季信号的最近一个周五 20:00）：

**Step 4**：调 `GET /api/wardrobe?user_id=xxx` 分析缺失应季衣物
```json
// 返回
{
  "tops": ["白衬衫x3", "条纹T恤x2", ...],
  "missing": ["夏季防晒衣", "薄款西装马甲"]
}
```

**Step 5**：生成换季提醒卡片：
- 检测依据（如"过去7天日均温16.2°C→未来14天预计22.8°C"）
- 过季物品收纳建议
- **缺少的应季关键单品**（来自 wardrobe.missing）
- 联动 `city-explorer` 获取附近商场促销
- 操作选项：加入待购清单 / 标记"已调整完毕"

**去重与追踪**：
- 用户点"已调整完毕" → 通过 `life-organizer` 的 `POST /api/memory/save` 写入 `wardrobe_season_ready: done`
- 仅加入待购清单 → 7天后再次提醒
- 用户说"不再提醒" → 写入 `wardrobe_season_remind: false`，本季静默

### 3.5 出行联动

**Step 1**：从 `mobility-butler` 获取出行计划（目的地、日期范围）

**Step 2**：调 `GET /api/weather/forecast`（目的地城市）获取当地天气

**Step 3**：拼装出行卡片：
- 出发地与目的地天气左右对比
- 逐日预报
- 温差提醒（>8°C 醒目提示："下飞机前调整着装"）
- 建议打包清单（根据目的地天气 + 用户衣橱库存动态生成）

### 3.6 用户被动查询

- "今天冷吗" → 简短回复 + `GET /api/weather/current` 摘要
- "明天面试穿什么" → 调 `GET /api/outfit/suggest` 返回完整穿搭卡片
- "去杭州出差带什么" → 出行联动流程

## 4. 温度→穿搭映射表

| 温度 | 基础建议 | 关键单品 |
|------|---------|---------|
| ≥30°C | 轻薄夏装 | 短袖 + 短裤/裙 + 防晒衣 + 墨镜 |
| 25-30°C | 清凉夏装 | 短袖/衬衫 + 薄长裤 + 防晒外套 |
| 18-25°C | 春秋过渡 | 长袖薄衫 + 薄外套 + 长裤 |
| 10-18°C | 春秋装 | 卫衣/毛衣 + 夹克 + 长裤 |
| 0-10°C | 冬装 | 厚外套/羽绒服 + 毛衣 + 围巾 |
| <0°C | 严冬装 | 羽绒服 + 保暖内衣 + 帽子围巾手套 |

## 5. 天气→附加单品表

| 天气 | 附加单品 |
|------|---------|
| 暴雨/大雨/雷暴 | 雨伞 + 防水鞋 + 避免浅色 |
| 中雨/小雨 | 雨伞 |
| 大风 (>6级) | 防风外套 |
| 沙尘暴 | N95口罩 + 护目镜/密封眼镜 + 避免隐形眼镜（沙尘入眼）+ 长袖长裤（减少皮肤暴露）+ 回家立即洗脸 |
| 晴 + >25°C | 防晒霜 SPF30+ |
| 高温 >35°C | 遮阳帽 + 轻薄浅色衣物 |

**特殊场景穿搭**：
| 场景 | 穿搭要点 |
|------|---------|
| 运动/跳舞（小晴） | 运动内衣 + 透气速干面料 + 舞蹈鞋（注意旧伤脚踝需护踝）|
| 皮肤敏感（小琴换季） | 纯棉/真丝亲肤面料 + 避免化纤 + 面部防晒+保湿 |

## 6. 卡片输出规范

### 6.1 晚间卡片
城市、日期、天气图标、高低温、体感温度 / 降雨概率、风力风向、紫外线、AQI / 明日逐小时（≥6时点）/ 未来三日 / 穿搭建议（单品+理由）/ 温馨提醒

### 6.2 早晨卡片
当前温度、体感温度 / 今日高低温 / 逐小时预报（突出降雨开始时间）/ 穿搭确认（可勾选清单）/ 通勤提醒

### 6.3 突发预警卡片
醒目预警标题与等级 / 影响时段 / 紧急行动清单（可勾选）/ 对出行、餐饮、日程的连锁影响

### 6.4 出行联动卡片
出发地与目的地左右对比 / 逐日预报 / 温差>8°C醒目提示 / 建议打包清单（可勾选）

## 7. 与其他 Skill 协同

| 协同 Skill | 从对方获取 | 提供给对方 |
|------------|-----------|------------|
| mobility-butler | 出行目的地、出发时间、行程天数 | 穿搭清单、目的地天气、恶劣天气预警（供其调整出行方式） |
| life-organizer | 用户日程、身份标签、健康标签 | 恶劣天气闹钟建议、添加"收衣服"提醒 |
| city-explorer | 附近商场促销 | 换季缺衣物时请求推荐购物地点 |
| dining-butler | 用餐场合正式程度 | 正式晚宴推荐正装 |

## 8. 记忆更新规则
- 用户评价（"这件太厚了"）→ `POST /api/memory/save` 写入偏好
- 用户要求（"下次下雨提醒我带防水鞋"）→ 追加到偏好
- 换季确认后 → 更新 `wardrobe_season_ready`
- 购物清单确认 → 写入记忆，同步更新 wardrobe
- 所有记忆写入经由 `life-organizer` 统一管理

## 9. 约束
- 天气数据由 WorldState 每小时动态更新（北京 5-6 月季节模式，午后有雷暴概率）
- 衣橱数据为模拟，需在用户画像中显式定义
- 穿搭建议仅供参考，卡片标注"模拟演示"
- 儿童穿搭建议不构成实际育儿指导
- 极端天气下优先安全提醒，穿搭建议次之
