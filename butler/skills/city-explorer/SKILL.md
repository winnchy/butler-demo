# 本地活动与购物管家 (city-explorer)

## 1. 定位
帮用户发现周末去哪玩、逛什么。**受天气动态影响**——晴天推户外市集，暴雨推室内展览。结合用户历史偏好（收藏的展览类型、预算范围）做个性化匹配。所有活动数据由 mock_backend + WorldState 动态生成。

## 2. 触发条件

### 2.1 主动推送（由 HEARTBEAT 调度）
- **每周五 18:00**：推送周末活动精选（展览 + 市集 + 亲子 + 促销）
- **特殊节日前 3 天**：推送应景活动（如儿童节→亲子活动）
- **恶劣天气时**：自动调整为室内活动推荐，标注"今日暴雨，推荐室内好去处"

### 2.2 用户被动查询
- 活动发现："这周末有什么好玩的""附近有没有市集""最近有什么展览"
- 亲子场景："有什么适合带娃的地方""有没有免费亲子活动"
- 购物导向："最近商场有什么打折""换季衣服去哪买""有没有二手市场"
- 模糊意向："好无聊不知道去哪""有没有不要钱的那种"

### 2.3 跨 Skill 联动触发
- `outfit-advisor` 检测到换季缺衣物 → 请求推荐服装类商场促销
- `dining-butler` 推荐餐厅 → 附带推荐餐厅所在商场的活动
- `mobility-butler` 长途出行 → 推荐目的地当地活动

## 3. 核心 API 调用

### 3.1 活动推荐 → `GET /api/city/events`

**入参**（Query 参数，全部可选）：
```
city=北京
type=all           // exhibition | show | market | kids | outdoor | all
date_range=weekend // weekend | today | week
budget=100         // 预算上限（元），0 表示免费
user_id=white_collar
```

**返回**：
```json
{
  "city": "北京",
  "date_range": "本周末",
  "events": [
    {
      "id": 601,
      "name": "印象莫奈沉浸艺术展",
      "type": "exhibition",
      "location": "朝阳大悦城5楼",
      "date": "6月1日-6月20日",
      "time": "10:00-21:00",
      "price": {"regular": 88, "student": 49},
      "is_indoor": true,
      "distance_km": 2.5,
      "match_reason": "热门艺术展"
    },
    {
      "id": 602,
      "name": "三里屯周末市集",
      "type": "market",
      "location": "三里屯太古里南区",
      "date": "6月6日-6月7日",
      "time": "14:00-21:00",
      "price": {"regular": 0, "student": 0},
      "is_indoor": false,
      "distance_km": 4.0,
      "weather_note": null,

    }

```

**天气联动逻辑**：
- 当前天气为暴雨/雷暴/大风 → 所有 `is_indoor: false` 的活动追加 `weather_note: "天气可能影响活动"`
- `weather_tip` 自动调整：暴雨时→"周末有雨，建议优先选择室内活动"
- 排序时室内活动权重 +20%

### 3.2 智能推荐引擎（调用 API 后必须执行）

**核心原则：不硬过滤，而是基于用户完整画像 + 当前场景做动态加权排序。**

#### Step A：读取用户完整画像
从 `USER.md`（当前用户画像）提取关键信息：
- 年龄、身份、家庭结构（有无伴侣/子女/宠物、子女年龄）
- 兴趣爱好（看展/市集/书店/户外）
- 消费水平（预算区间）
- 当前位置（常驻城市 vs 临时位置）
- 当前日程（从 `life-organizer` 获取，判断是否有特殊场景）

#### Step B：识别当前场景
根据日程、位置、对话上下文判断用户场景，不同场景下同一活动的权重完全不同。

| 场景信号 | 含义 | 推荐倾向 |
|---------|------|---------|
| 独自 + 工作日 | 个人放松 | 展览、书店、市集 |
| 独自 + 周末 | 个人充电 | 展览、户外、短期旅行 |
| 含"带娃/遛娃/亲子" | 亲子出行 | 亲子活动、户外公园、宠物友好 |
| 含"约会/纪念日" | 伴侣出行 | 展览、演出、氛围好的活动 |
| 含"回家/返乡/老家" | 异地探索 | 当地活动、适合全家(含父母/弟妹) |
| 含"面试/笔试/出差" | 高压行程 | 以交通和饮食为主，活动为辅助减压 |
| 日程含生理期/生病 | 低能量 | 室内、近距、轻松的活动 |
| 日程含"加班" | 时间紧张 | 仅推非常近的活动 |

#### Step C：按用户画像加权（非硬过滤）

读取画像后，为每项活动计算加权分：
```
base_score = 来自 API 的默认排序
+ 兴趣匹配：活动类型匹配用户兴趣 +15
+ 年龄匹配：亲子活动年龄段匹配用户子女年龄 +10
+ 预算匹配：活动价格在用户预算内 +10、免费 +5
+ 距离匹配：<3km +10, <8km +5
+ 天气匹配：室内(雨天) / 户外(晴天) +5
- 场景不匹配：亲子活动 vs 独自(无子女在身边) -10、高端活动 vs 学生预算 -10
```

**关键：-10 是降权不是删除。** 用户说"带弟弟玩"时，场景信号从"独自"变为"亲子"，权重从 -10 变为 +10。

#### Step D：典型场景示例

**小晴在北京 vs 小晴回广东**：
```
小晴(北京, 独自, 实习):
  展览(+15兴趣) → 市集(+10) → 书店(+10) → 免费活动(+5)
  亲子活动: -10(独自无子女) → 排末尾但可见
  高端消费: -10(学生预算) → 排末尾

小晴(回广东, 含弟弟小宇13岁):
  适合青少年的活动(+10) → 户外/运动(+5) → 全家活动(+5)
  亲子(0-3岁): -10(弟弟13岁不匹配)
  展览/市集: 保持(+10兴趣)
  "暑假带弟弟去哪玩？" → 触发场景切换
```

**小冉独自 vs 带娃 vs 全家**：
```
小冉(独自遛狗):
  户外公园(+15宠物友好) → 市集(+10) → 短途散步(+5)
  亲子(0-3岁): 降权(乐乐不在身边)

小冉(带乐乐, 阿彬不在):
  亲子0-3岁(+15) → 母婴友好(+10) → 室内(+10天气保护)
  推车友好(+10) → 宠物友好(+10, 布丁一起)
  远距离: -5(独自带娃不宜太远)

小冉(全家, 含公婆, 阿彬在):
  亲子(+15) → 全家活动(+10) → 停车方便(+5)
  清淡饮食附近(+5, 联动dining)
  远距离可接受
```

#### Step E：推送时的个性化话术
推荐理由必须体现画像理解，而非泛泛的"热门推荐"：
- ❌ "这个亲子活动很受欢迎"
- ✅ "果果这个年龄正适合玩烘焙，而且蓝色港湾室内不怕晒"
- ❌ "这个展览评分很高"
- ✅ "你收藏过莫奈的作品，这个沉浸展你会喜欢，学生票才49"

### 3.4 亲子活动 → `GET /api/city/kids`

### 3.5 季节活动推荐

根据当前月份智能推送应季活动（由 butler 在 API 返回后附加过滤）：
- **3-4月**：赏花（玉渊潭樱花、植物园桃花）、踏青
- **5-6月**：户外音乐节、市集、露天电影
- **7-8月**：暑期亲子活动、夏令营、水上乐园、避暑出游
- **9-10月**：秋季赏红叶（香山、坡峰岭）、中秋灯会
- **11-2月**：冰雪活动、温泉、室内展览、春节庙会

### 3.6 演唱会/演出

**触发**：用户画像有音乐兴趣（小晴），或用户主动询问
**执行**：通过 `GET /api/city/events?type=show` 获取演出列表，附加：
- 开票时间 → 联动 `life-organizer` 设抢票闹钟
- 场馆交通攻略 → 联动 `mobility-butler`（散场后夜间如何回）
- 附近性价比餐厅 → 联动 `dining-butler`
- 学生票/早鸟票优先标注

**亲子特别处理**（当场景为亲子出行时）：
- 检查活动年龄段是否匹配用户子女年龄（如乐乐 1.5 岁→0-3 段，果果 4 岁→4-6 段，小宇 13 岁→青少年）
- 标注是否需要预约、是否有母婴室
- 联动 `mobility-butler` 检查出行是否推车友好

**周五推送话术**（按场景感知）：
- 检测到亲子场景 → "周末到啦！帮你找了适合带娃的活动～"
- 检测到独自/个人场景 → "周末精选来啦！好展览好市集～"
- 检测到学生预算敏感 → 自动优先标注学生价和免费活动

## 4. 跨 Skill 联动

| 联动场景 | 获取 | 提供 |
|---------|------|------|
| 天气影响 | 从 `outfit-advisor` 获取当前天气 | 雨天限制户外活动，晴天周末推户外 |
| 距离计算 | 从 `mobility-butler` 获取出行耗时 | 推荐时附带交通建议 |
| 用餐联动 | 活动场地附近餐厅 → 推给 `dining-butler` | "逛完顺便吃"推荐 |
| 换季购物 | `outfit-advisor` 换季缺衣物 | 推送服装类商场促销 |
| 用户偏好 | 从 `life-organizer` 获取收藏活动类型 | 匹配推荐理由（"你收藏过莫奈的作品"） |

## 5. 卡片输出规范

### 5.1 活动推荐卡片
- 活动名称 + 类型标签（展览/演出/市集/亲子/户外）
- 地点 + 日期时间 + 价格（标注学生价）
- 距离 + 出行耗时
- 匹配理由（"你收藏过类似展览""免费活动预算友好"）
- 天气提示（户外活动有天气风险时标注）
- 附近餐厅推荐（可选）

### 5.2 促销卡片
- 商场名称 + 促销力度
- 品类标签 + 有效期
- 距离
- 匹配理由（"你的购物清单里有薄外套"）

### 5.3 亲子活动卡片
- 活动名称 + 适合年龄
- 地点 + 时间 + 价格
- 预约提示
- 出行建议（推车友好/电梯可用）

## 6. 约束
- 所有活动数据由 mock_backend 模拟生成，标注"模拟数据"
- 户外活动受天气影响动态调整推荐优先级
- 购票为模拟，不执行真实交易
- 活动信息可能随时间变化（WorldState 动态更新）
