### 场景1：去上海终面 — 飞机延误 → 延误险 → 高铁紧急替代（极限时间链）
**触发时间**：某个周日下午
**触发条件**：小晴收到某大厂通知：后天（周二）上午10:00 上海终面。必须到现场。Mock 后端生成"航班延误"事件。

**事件链**：
1. **周日 16:00** 收到面试通知 → `mobility-butler` 启动长途出行规划（阶段1-2）：
   - 目的地：上海，后天上午10:00前必须到达面试地点（浦东某写字楼）
   - 方式：飞机（北京→上海，约2.5小时）+ 落地后地铁/打车到面试地点
   - 推送周一晚间或周二早班航班（早班太赶，建议周一晚上到，住一晚）
2. **订票提醒**：`mobility-butler` 展示周一 18:00-21:00 三个航班选项，含模拟价格（特价票 vs 全价票）。小晴选最便宜的特价票（学生预算）。**明确标注"模拟数据，请自行购票"**
3. **天气预警**：`outfit-advisor` 拉取周一北京+上海天气 → **北京周一晚间有雷暴预警！** → 立即推送：
   - "小晴，周一晚上北京有雷暴，你选的那班特价航班准点率只有40%。强烈建议：①买延误险（约30元）②同时查高铁备选（北京南→上海虹桥，约4.5小时，最晚一班18:00）"
   - `mobility-butler` 并行展示"飞机 vs 高铁"对比卡片：耗时、费用、确定性、到上海后的交通
4. **面试穿搭**：`outfit-advisor` 推送上海天气（28-33°C，湿热）+ 面试穿搭（西装外套+黑裤+乐福鞋，建议带一件备用衬衫防止出汗）。提醒风衣扣子还没缝好！（从 wardrobe missing 读取）
5. **周一 16:00 果然延误了**（Mock 触发）：航班延误2小时 → `mobility-butler` 立即推送：
   - **主线程**："航班延误2小时，预计21:00才能起飞，到上海已凌晨。面试在明早10:00，睡眠不足会影响状态。推荐：立即改高铁！北京南→上海虹桥 G17 18:00发车 22:30到。现在出发去北京南站：打车约40分钟（费用约65元），地铁约55分钟（费用6元）。建议打车（行李+面试正装不便挤地铁）"
   - **并行线程**：
     - 延误险 → "本次购票时听了建议买了延误险（30元），延误超2小时可理赔300元。记得在App上申请，保留登机牌截图"
     - 退票 → "特价票因天气原因延误，可免费退票（模拟提醒）"
     - 酒店 → "高铁22:30到上海，推送面试地点附近3家经济型酒店（预算200-350），建议选步行可达面试地点的"
     - 面试准备 → "高铁4.5小时，正好在车上复习面试内容。推送：WiFi密码、餐车位置、到站提醒"
6. **周二 7:00 上海**：`outfit-advisor` 推送上海当日天气+面试穿搭确认。`mobility-butler` 推送酒店→面试地点的步行/骑行/打车方案
