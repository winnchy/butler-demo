### 场景1：工作日接待上级午餐（全流程联动）
**触发时间**：某个工作日上午10:00
**触发条件**：小琴日程中有"12:00 接待王总+深圳来的合作方（3人）"

**事件链**：
1. **10:00** heartbeat 检测到日程 → 唤醒 `dining-butler`："今天中午接待王总和深圳合作方，共3人，帮我找个合适的餐厅"
2. **餐厅筛选**：`dining-butler` 分析需求 — 王总不吃辣、偏好清淡；合作方来自深圳（粤菜偏好？）；需包厢、安静、步行可达或短途。推荐3家并排序（鼎泰丰包厢/粤来越好/将太无二），附理由。
3. **未知忌口处理**：合作方的口味未知 → `dining-butler` 建议小琴委婉打听（"深圳那边口味偏淡，粤菜应该合适，但如果有特别忌口提前跟我说"），同时准备备选方案（点菜时留2道清淡菜）
4. **提前取号**：选定餐厅后 → `dining-butler` 远程取号/预约包厢 → 开启排队监控
5. **出行规划**：`mobility-butler` 规划望京→餐厅路线（中午时段，步行 vs 打车）。今日若下雨 → 建议打车。若限行 → 提醒小琴今天不能开车。
6. **出发提醒**：11:35 预估步行12分钟 → 推送"现在出发可提前5分钟到，在包厢等客人更从容"
7. **到店异常**：11:48 到店后发现包厢还在收拾，需等15分钟 → `dining-butler` 推送应急方案：
   - 方案A：大堂临时卡座先喝茶（推荐，不折腾）
   - 方案B：隔壁同商场另一家粤菜馆有包厢空位（步行2分钟）
   - 小琴选A → `life-organizer` 记下"鼎泰丰包厢需提前确认收拾好"
8. **餐后**：`dining-butler` 更新王总和合作方口味档案；`life-organizer` 记录本次接待要点
