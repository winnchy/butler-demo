# 正确架构：OpenClaw 为核心 + 阿里云部署方案

## 一、你需要的正确架构

```
┌─────────────────────────────────────────────────────────┐
│  阿里云轻量应用服务器                                      │
│                                                         │
│  ┌──────────────────────────────────────────────┐       │
│  │  Docker: OpenClaw v2026.4.15                  │       │
│  │  - "脑子" = Anthropic API Key（环境变量注入）   │       │
│  │  - "技能书" = butler/ 下的所有 SKILL.md        │       │
│  │  - "记忆" = USER.md + MEMORY.md（切换用户时换） │       │
│  │  - "性格" = SOUL.md                           │       │
│  │  - 需要数据时 → 调 http://localhost:8000/api/* │       │
│  │  - 对外暴露 Web 聊天界面（或 API）              │       │
│  └──────────────────────────────────────────────┘       │
│                      ↕                                   │
│  ┌──────────────────────────────────────────────┐       │
│  │  FastAPI (main.py) :8000                      │       │
│  │  - 所有 mock 数据 API                          │       │
│  │  - WorldState 后台引擎（天气/路况/排队持续变）   │       │
│  │  - 管理面板 /admin/dashboard                   │       │
│  └──────────────────────────────────────────────┘       │
│                      ↕                                   │
│  ┌──────────────────────────────────────────────┐       │
│  │  前端聊天界面（HTML，FastAPI 直接 serve）       │       │
│  │  - 评委打开链接 → 看到管家 → 自由对话            │       │
│  │  - 语音输入（浏览器 Web Speech API）            │       │
│  │  - 对话请求发给 OpenClaw，OpenClaw 调 API 回复  │       │
│  └──────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────┘
```

### 数据流

```
评委输入 "我下周二去深圳面试"
  → 前端 POST 到 OpenClaw 的对话接口
    → OpenClaw 读 SOUL.md（管家性格：预判型、面面俱到）
    → OpenClaw 读 USER.md（小晴：研一、秋招中、深圳是老家、生活费5000）
    → OpenClaw 用 LLM 推理：
        "用户要去深圳面试 →
         ① 查北京→深圳机票（mobility.transport/search）
         ② 查深圳天气 vs 北京温差（weather/forecast）
         ③ 规划面试穿搭（outfit/suggest，场景=interview）
         ④ 面试地点附近酒店（mobility/nearby）
         ⑤ 加日程提醒（schedule/create）
         ⑥ 检查有没有延误险需求（mobility 长途规划）"
    → OpenClaw 依次调 mock_backend API 获取数据
    → OpenClaw 拼接回复："帮你看了周二北京飞深圳的航班……"
  → 前端展示管家回复（卡片形式）
```

## 二、OpenClaw + Mock Backend 的连接方式

OpenClaw 的 SKILL.md 中定义了每个技能"调什么 API"。
我们的 mock_backend 提供了这些 API。
把它们跑在同一台服务器上，OpenClaw 调 `localhost:8000` 就行。

```
OpenClaw SKILL.md 写的：           Mock Backend 实际提供：
POST /api/dining/recommend    →    dining_routes.py 的 recommend()
GET  /api/weather/current     →    outfit_routes.py 的 current_weather()
POST /api/mobility/route      →    mobility_routes.py 的 plan_route()
...                                  ...
```

**部署时**：OpenClaw 的配置文件里设 `API_BASE_URL=http://localhost:8000`，所有 Skill 的 API 调用就自动指向 mock_backend。

## 三、什么是 Docker

Docker 就是一个"打包工具"。类比：

```
没有 Docker：
  你在阿里云买台服务器 → 装 Ubuntu → 装 Python 3.12 → 装 Node.js 20
  → 装各种依赖 → 配环境变量 → 下载 OpenClaw → 启动
  每一步都可能出错，换一台服务器重来一遍

有了 Docker：
  你把 OpenClaw + 所有依赖打包成"镜像"
  服务器上只需要装 Docker 这一个东西
  然后 docker run 你的镜像 → 跑起来了
  换服务器？同样的镜像直接跑，环境一模一样
```

OpenClaw 官方提供了 Docker 镜像，你不需要自己打包。只需要：
```bash
# 在服务器上
docker pull openclaw/openclaw:v2026.4.15
docker run -d \
  -v /path/to/butler:/app/workspace \    # 挂载你的 butler 目录
  -e ANTHROPIC_API_KEY=sk-ant-xxx \      # 注入 LLM API Key
  -p 3000:3000 \                          # 暴露端口
  openclaw/openclaw:v2026.4.15
```

## 四、前端怎么办

OpenClaw 自带了 Web 聊天界面（通过其 Gateway 模块）。评委打开 `http://你的服务器IP:3000` 就能看到聊天界面。

如果你想要更定制化的界面（logo、品牌色、场景按钮），就自己写一个薄的前端，通过 OpenClaw 的 API 或 WebSocket 跟它通信。

**建议**：先用 OpenClaw 自带的 Web 界面演示，省时间。如果有多余精力再定制。

## 五、部署步骤（阿里云方案）

### 1. 购买服务器
- 阿里云轻量应用服务器，2核2G，约 68元/月
- 学生认证后可能更便宜（送 300 元券）
- 选 Ubuntu 22.04 系统

### 2. 装 Docker
```bash
curl -fsSL https://get.docker.com | sh
```

### 3. 把项目上传到服务器
```bash
# 用 git clone 或 scp 上传整个 mt_workspace
git clone <你的仓库地址>
cd mt_workspace
```

### 4. 启动 Mock Backend
```bash
cd mock_backend
pip install fastapi uvicorn
python enrich_pois.py
python route_generator.py
nohup python main.py &    # 后台运行，端口 8000
```

### 5. 启动 OpenClaw
```bash
docker run -d --name butler-openclaw \
  -v $(pwd)/butler:/app/workspace \
  -e ANTHROPIC_API_KEY=你的key \
  -e API_BASE_URL=http://localhost:8000 \
  -p 3000:3000 \
  openclaw/openclaw:v2026.4.15
```

### 6. 配 Nginx（可选，为了好看）
```bash
# 配一个反向代理，让评委直接访问 http://你的域名 就能用
# 不加也行，直接访问 http://IP:3000
```

### 7. 设置 API Key（安全）
```bash
# 在服务器上创建 .env 文件（不提交 Git）
echo "ANTHROPIC_API_KEY=sk-ant-xxx" > /etc/openclaw/.env
# Docker 启动时读取这个文件
```

## 六、LLM API Key 怎么办

你需要一个 Anthropic API Key（Claude）。获取方式：
1. 去 console.anthropic.com 注册
2. 创建 API Key
3. 充值（Claude 对话成本很低，比赛演示用不了几块钱）

**Key 安全**：
- `.env` 文件放服务器上，不提交 GitHub
- `.gitignore` 加 `.env`
- Docker 通过环境变量注入，不硬编码

## 七、没有服务器怎么办（Plan B）

如果暂时搞不定阿里云，本地也能演示：
1. 自己电脑上跑 mock_backend（`python main.py`）
2. 自己电脑上 Docker 跑 OpenClaw
3. 用 **ngrok** 暴露外链：`ngrok http 3000` → 得到一个临时公网 URL
4. 评委访问这个 URL 就能用
5. 缺点：电脑不能关、URL 每次变
